# Protobuf Contracts

Protobuf is **central** to this architecture. Every service interface is
defined in proto, and the custom `protoc-gen-go-contracts` plugin generates
the Go application-layer contract (interface + noop stub + event topics +
error codes) automatically. You write the proto once; everything else is
generated.

## Two-Level Generation

`buf generate` runs two layers of plugins for each domain:

```
contracts/proto/todo/v1/todo.proto
         │
         ├── buf.build/protocolbuffers/go
         │       └── go/network/todo/v1/todo.pb.go          ← message structs
         │
         ├── buf.build/connectrpc/go
         │       └── go/network/todo/v1/todov1connect/       ← handler interface + client
         │
         └── protoc-gen-go-contracts  (custom plugin)
                 └── go/application/todo/
                     ├── todo_service_contract_gen.go        ← TodoService interface + NoopTodoService
                     ├── events_gen.go                       ← topic constants + type aliases
                     └── errors_gen.go                       ← error code constants
```

The network layer (`go/network/…`) is for transport. The application layer
(`go/application/…`) is for in-process contract wiring.

## Transport Swapping

The key advantage of combining Connect RPC with the contract pattern:
the same inbound adapter serves both in-process and network callers.

**In-process (default in the monolith):**

```
cmd/mmw/main.go
  └── authModule.PrivateService()
        └── calls auth application service methods directly (no network, no serialisation)
```

**Over-the-network (when splitting to microservices):**

```
todo process                           auth process
  └── defauth.NewPrivateHTTPClient       └── Connect HTTP server
        └── authv1connect.NewAuthPrivateServiceClient  └── AuthPrivateHandler (same code)
              └── HTTP/2 + protobuf
```

The `AuthHandler` Connect implementation changes nothing. Only the
composition root swaps `authModule.PrivateService()` for a
`defauth.NewPrivateHTTPClient(connectClient)`.

## Connect Handler Structure

The generated interface (from `buf.build/connectrpc/go`) that the inbound adapter implements:

```go
// Generated by buf: contracts/go/network/todo/v1/todov1connect/todo.connect.go
package todov1connect

type TodoServiceHandler interface {
    CreateTodo(context.Context, *connect.Request[todov1.CreateTodoRequest]) (*connect.Response[todov1.CreateTodoResponse], error)
    GetTodo(context.Context, *connect.Request[todov1.GetTodoRequest]) (*connect.Response[todov1.GetTodoResponse], error)
    CompleteTodo(context.Context, *connect.Request[todov1.CompleteTodoRequest]) (*connect.Response[todov1.CompleteTodoResponse], error)
    ReopenTodo(context.Context, *connect.Request[todov1.ReopenTodoRequest]) (*connect.Response[todov1.ReopenTodoResponse], error)
    DeleteTodo(context.Context, *connect.Request[todov1.DeleteTodoRequest]) (*connect.Response[todov1.DeleteTodoResponse], error)
    ListTodos(context.Context, *connect.Request[todov1.ListTodosRequest]) (*connect.Response[todov1.ListTodosResponse], error)
}

func NewTodoServiceHandler(svc TodoServiceHandler, opts ...connect.HandlerOption) (string, http.Handler)
```

The adapter implements this interface and is registered via `NewTodoServiceHandler`.

## Application-Layer Contract (generated by `protoc-gen-go-contracts`)

The custom plugin generates the Go interface from the proto `service` block,
plus a `NoopTodoService` stub and a sentinel error:

```go
// contracts/go/application/todo/todo_service_contract_gen.go
// Code generated by protoc-gen-go-contracts. DO NOT EDIT.
package todo

import (
    "context"
    "fmt"
    todov1 "github.com/pivaldi/mmw-contracts/go/network/todo/v1"
)

type TodoService interface {
    CreateTodo(ctx context.Context, req *todov1.CreateTodoRequest)   (*todov1.CreateTodoResponse, error)
    GetTodo(ctx context.Context, req *todov1.GetTodoRequest)         (*todov1.GetTodoResponse, error)
    // … one method per proto service RPC
}

var ErrTodoServiceUnavailable = fmt.Errorf("todo service unavailable")

type NoopTodoService struct{}
var _ TodoService = (*NoopTodoService)(nil)

func (NoopTodoService) CreateTodo(_ context.Context, _ *todov1.CreateTodoRequest) (*todov1.CreateTodoResponse, error) {
    return nil, ErrTodoServiceUnavailable
}
// … one method per interface method
```

## Event Topics (generated)

```go
// contracts/go/application/todo/events_gen.go
// Code generated by protoc-gen-go-contracts. DO NOT EDIT.
package todo

import todov1 "github.com/pivaldi/mmw-contracts/go/network/todo/v1"

const (
    TopicUserTaskCreated   = "todo.userTasks.created.v1"
    TopicUserTaskUpdated   = "todo.userTasks.updated.v1"
    TopicUserTaskCompleted = "todo.userTasks.completed.v1"
    TopicUserTaskReopened  = "todo.userTasks.reopened.v1"
    TopicUserTaskDeleted   = "todo.userTask.deleted.v1"
    TopicUserTasksDeleted  = "todo.userTasks.deleted.v1"
)

var Topics = []string{
    TopicUserTaskCreated, TopicUserTaskUpdated, TopicUserTaskCompleted,
    TopicUserTaskReopened, TopicUserTaskDeleted, TopicUserTasksDeleted,
}

// Type aliases re-export proto types under the application package.
type UserTaskCreatedEvent   = todov1.UserTaskCreatedEvent
type UserTaskCompletedEvent = todov1.UserTaskCompletedEvent
```

## Connect vs gRPC-Gateway

This architecture uses [`connectrpc.com/connect`](https://connectrpc.com),
not the gRPC-Gateway pattern. Connect differences:

| | Connect | gRPC-Gateway |
|---|---|---|
| Transport | HTTP/1.1 + HTTP/2 | HTTP/2 only (gRPC) |
| Browser clients | Native (no proxy) | Requires Envoy proxy |
| Generated code | `buf.build/connectrpc/go` | `buf.build/grpc-ecosystem/gateway` |
| Interceptors | `connect.Interceptor` | gRPC middleware |

## Generating Contracts

```bash
cd poc/contracts

# 1. Build the custom plugin (once, or after plugin code changes):
go build -o $(go env GOPATH)/bin/protoc-gen-go-contracts ./cmd/protoc-gen-go-contracts

# 2. Generate for each domain:
buf generate --template buf.gen.todo.yaml
buf generate --template buf.gen.auth.yaml
```

Each `buf.gen.*.yaml` runs all three plugins (protobuf, connectrpc, go-contracts)
and outputs to `go/network/…` (transport types) and `go/application/…` (contract interfaces).
